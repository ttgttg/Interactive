Stage 2 Layout and Structure

Date: 14 March 2026

I planned the structure of the website and experimented with layout options. At this stage, I focused on navigation, and did some inform user testing during class by letting others simulate and make assumptions about how they would navigate the site.

I didn't manage to flesh out all the bits I wanted to explore, and my design pivoted from a scrapbook-like feel to a more cyber-inspired look. I was inspired by the use of getUserMedia() in the interactive media webpage and wanted to experiment using this function.

An attempt with Google Stitch

Date: 14 March 2026

Tried experimenting with stitch to vibe code the website - but honestly not too satisfied with the output. I need to look into prompt engineering to manipulate it better.
I feel like I was negotiating with the tool instead of having direct dictation over it. What felt awkward at first was the unpredictability and the lack of precision - I'm not used to articulating my design thoughts through prompts. It also struggled to capture the specific aesthetic I was aiming for — especially the darker, surveillance-style interface - often defaulting to cleaner, more generic UI patterns.
That said, what worked really well was how quickly it could generate structured layouts and visual directions. It was surprisingly good at producing coherent systems — things like navigation hierarchies, modular panels, and consistent UI components.
Overall, I'm keen to keep experimenting with stitch and learn more about how to guide it.

Web design choices

Date: 25 March 2026

Fonts

For this project, I chose a monospaced, terminal-style font to reinforce the system/surveillance aesthetic. I’m currently using something like VT323, Special Elite Monospace, and Lucida Console. The first two are static fonts, sourced from Google Fonts and linked directly into the HTML using an @import or tag. I chose this approach because it’s lightweight and easy to integrate.

Website's Narrative

  • interface language (e.g. “Access Verification”, “Initiate Identity Scan”)
  • system feedback loops (camera access, scanning, confirmation)
  • subtle environmental cues like glitches and overlays

The point is that the user is being constantly monitored, and the website would respond subtly to every interaction they make, in an uncanny way.
I was interested in capturing a subtle discomfort of being observed, processed, and not fully in control of the environment.
